# Education Prompt Skill 工具实现计划

## 1. 项目概述
将 `/Users/hulimofaqiu/Documents/obisidian笔记文件/education- prompt.md` 转换为 Claude Agent Skill 工具，实现教育场景下的多种AI辅助功能。

## 2. 核心功能与适用场景
该Skill将包含沃顿商学院教授分享的20+种教育类系统提示词，主要分为学生端和教师端（或通用教育场景）功能：

### 学生端功能
- 模拟场景创建器（Simulation Creator）
- 课程项目想法生成器
- 测验创建器（Quiz Creator）
- 主动学习协作者
- 教学大纲协作者
- 主题解释协作者
- 结构化提示词设计器
- 课程内容构建器
- 通用导师
- AI反馈导师
- AI学生模拟器
- 谈判模拟器
- 团队行动后回顾
- 团队章程生成器
- 课堂反思助手
- 魔鬼代言人

## 3. 技术实现方案
### Skill文件结构
遵循标准Claude Agent Skill结构，包含：
- YAML Frontmatter（技能名称、描述、许可证、允许使用的工具等）
- Markdown内容（功能介绍、使用说明、提示词模板等）

### 提示词组织策略
将原始文件中的各类提示词进行分类整理，每种功能对应一个明确的提示词模板，并提供使用示例。

### 工具集成
根据需要允许使用的工具：
- Write: 生成教学材料
- WebSearch: 搜索教育资源
- Task: 调用其他技能协作
- Bash: 执行简单脚本（可选）

## 4. 实现步骤
### 步骤1：创建基础Skill结构
- 复制已验证的`frontend-design.md` Skill文件结构
- 配置YAML Frontmatter
- 确保文件权限正确

### 步骤2：整理提示词内容
- 从原始文件中提取所有教育类提示词
- 分类并优化每个提示词的格式
- 为每个功能编写使用说明和示例

### 步骤3：实现核心功能逻辑
- 在Skill中实现专家召唤和提示词生成逻辑
- 确保双语支持（英文原始提示词 + 中文说明）

### 步骤4：测试与优化
- 在本地环境测试Skill功能
- 根据测试结果调整提示词和功能逻辑

## 5. 预期成果
- 一个功能完整的`education-prompt-assistant.md` Skill文件
- 20+种教育场景下的AI辅助功能
- 详细的使用说明和示例
